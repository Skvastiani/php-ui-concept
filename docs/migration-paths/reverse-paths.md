
# Reverse Migration Paths: PHP to Other Languages

## Purpose

Reverse paths are included to make the project credible and transparent. The site should not trap users inside PHP marketing. It should help developers make informed technical choices.

Required routes:

```txt
/paths/php-to-javascript
/paths/php-to-python
/paths/php-to-go
/paths/php-to-java
/paths/php-to-csharp
/paths/php-to-ruby
```

First prototype requirement:

- Create teaser cards or placeholder pages.
- Link to them from every `<language>-to-php` page.
- Use honest positioning.

Example copy:

```txt
Thinking about moving the other way?
Explore PHP → Go and understand when Go may be a better fit.
```

## Required Reverse Page Sections

- Why leave PHP for this language?
- What problems this language solves better than PHP
- What you may lose when leaving PHP
- Syntax comparison
- Ecosystem/tooling comparison
- Third-party benchmarks
- Contribution CTA

## Strategic Reason

Helping users leave creates trust. It shows the project is about developer decision quality, not shallow promotion.
